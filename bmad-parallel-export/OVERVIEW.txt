📖 KEY MODIFICATIONS FOR PARALLEL DEVELOPMENT
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

ARCHITECT
─────────
  • Layer-Based Sharding: Divides architecture into -frontend.md and -backend.md
  • API-First Contracts: Strictly defines API contracts before implementation

PRODUCT MANAGER & PRODUCT OWNER  
────────────────────────────────
  • Story Parallelization: Creates paired Backend (API/DB) and Frontend (UI/UX)
    stories that are independent and can be developed in parallel

DEVELOPER-FRONT
───────────────
  • Frontend-only scope: Implements ONLY frontend stories
  • Contract-driven mocks: Uses API contracts to create mocks
  • Reads ONLY -frontend.md shards

DEVELOPER-BACK
──────────────
  • Backend-only scope: Implements ONLY backend stories
  • API contract fidelity: Delivers APIs exactly as architected
  • Reads ONLY -backend.md shards

The install-bmad-parallel.sh script automatically:

  1. Detects BMAD directory structure (.bmad-core/ or .bmad/)
  2. Creates directory if it doesn't exist
  3. Backs up existing agent files with timestamps
  4. Downloads all 5 custom agent files (curl or wget)
  5. Installs files into target project
  6. Reports success/failure with colored output
  7. Provides backup location and summary

Requirements:
  • Bash shell
  • curl or wget
  • Write permissions in target directory
