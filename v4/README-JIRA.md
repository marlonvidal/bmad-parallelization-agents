# Jira Integration for BMAD Parallel Agents

## Overview

This document explains how to use the optional Jira integration with BMAD Parallel Agents. With Jira integration, you can automatically sync PRD epics and stories to Jira for project tracking, keeping your development work and project management in sync.

## Key Features

✅ **Optional Integration** - Works perfectly without Jira, zero impact on core workflows  
✅ **One-Way Sync** - BMAD documents are source of truth, Jira is a mirror  
✅ **Hierarchical Sync** - PRD Epic → Jira Epic, PRD Story → Jira Story, Story Tasks → Jira Tasks  
✅ **Both PM and SM Agents** - Sync command available in both agents (no context switching)  
✅ **Incremental Updates** - SM updates Jira when drafting stories with detailed tasks  
✅ **Auto-Detection** - Automatically detects if Jira MCP is configured

## Quick Start

### 1. Prerequisites Setup

Before using Jira integration, complete the setup in [JIRA-SETUP.md](JIRA-SETUP.md). This includes:

- Creating Jira API token
- Installing Jira MCP server
- Configuring `~/.cursor/mcp.json`
- Updating `.bmad-core/core-config.yaml`

**Time Required**: 15-20 minutes (one-time setup)

### 2. Verify Setup

```bash
# Check MCP is configured
cat ~/.cursor/mcp.json | grep jira

# Check core config has jira section
cat .bmad-core/core-config.yaml | grep -A 10 "jira:"
```

### 3. Test Integration

Open Cursor in your project and test:

```
@pm
*help
```

You should see `*sync-to-jira` command in the list.

## Usage Workflows

### Workflow 1: Sync PRD After Creation (PM Agent)

**Scenario**: You've just created a PRD and want to create Jira tracking items.

```
@pm

*create-prd
# ... create your PRD ...

# At the end, PM will detect Jira MCP and ask:
# "Would you like to sync this PRD to Jira now? (Y/n)"

# If you choose Yes:
# - Epics created in Jira
# - Stories created and linked to epics
# - Mapping file saved
# - Jira links displayed
```

**Result**:
- All PRD epics created as Jira Epics
- All PRD stories created as Jira Stories (linked to parent epics)
- Mapping file created at `.bmad-core/data/jira-mapping.yaml`
- You receive clickable Jira links

### Workflow 2: Sync Existing PRD (PM or SM Agent)

**Scenario**: You have an existing PRD that wasn't synced to Jira.

```
@pm
*sync-to-jira

# Or from SM agent:
@sm
*sync-to-jira

# Agent will:
# 1. Check Jira MCP availability
# 2. Load PRD content
# 3. Create epics in Jira
# 4. Create stories in Jira
# 5. Save mapping file
# 6. Display summary with links
```

**Result**: Same as Workflow 1, but manually triggered.

### Workflow 3: Draft Story with Jira Updates (SM Agent)

**Scenario**: You're drafting the next story and want to update Jira with details.

```
@sm
*draft

# SM will:
# 1. Create story file with full details
# 2. Check if jira-mapping.yaml exists
# 3. If mapping exists:
#    - Update Jira story description with full story details
#    - Create Jira tasks for each story subtask
#    - Add Jira links to story file
# 4. If mapping doesn't exist:
#    - Ask if you want to sync PRD now
#    - If yes, run *sync-to-jira automatically
```

**Result**:
- Story file created with all technical details
- Jira story updated with comprehensive description
- Jira tasks created for each story subtask
- Story file includes Jira links section

### Workflow 4: Sync Without Jira (Default Behavior)

**Scenario**: Jira MCP is not configured (or you choose to skip).

```
@pm
*create-prd
# ... create PRD ...

# No Jira prompt appears
# PRD creation completes normally
```

**Result**: Everything works as usual, no Jira integration.

## Command Reference

### PM Agent Commands

| Command | Description | When to Use |
|---------|-------------|-------------|
| `*create-prd` | Create PRD with optional Jira sync | Starting new project |
| `*sync-to-jira` | Sync PRD to Jira | After PRD creation or updates |

### SM Agent Commands

| Command | Description | When to Use |
|---------|-------------|-------------|
| `*draft` | Create next story with optional Jira updates | Creating each story |
| `*sync-to-jira` | Sync PRD to Jira (same as PM) | If mapping doesn't exist |

## Data Structures

### Jira Mapping File

**Location**: `.bmad-core/data/jira-mapping.yaml`

**Purpose**: Tracks relationships between BMAD documents and Jira issues

**Structure**:
```yaml
projectKey: PROJ
baseUrl: https://your-company.atlassian.net
syncedAt: 2026-02-25T10:30:00Z
epics:
  1: PROJ-100
  2: PROJ-101
  3: PROJ-102
stories:
  "1.1": PROJ-103
  "1.2": PROJ-104
  "2.1": PROJ-105
tasks:
  "1.1.1": PROJ-106
  "1.1.2": PROJ-107
  "1.2.1": PROJ-108
```

**Notes**:
- Auto-generated by `*sync-to-jira` command
- Updated when SM drafts stories (adds tasks)
- Human-readable for debugging
- Consider adding to `.gitignore` if Jira IDs are sensitive

### Story File Jira Links Section

**Location**: Added to story files (e.g., `docs/stories/1.1.setup-auth.md`)

**Example**:
```markdown
## Jira Links
- Epic: [PROJ-100](https://your-company.atlassian.net/browse/PROJ-100)
- Story: [PROJ-103](https://your-company.atlassian.net/browse/PROJ-103)
- Tasks:
  - [PROJ-106](https://your-company.atlassian.net/browse/PROJ-106) - Setup database schema
  - [PROJ-107](https://your-company.atlassian.net/browse/PROJ-107) - Create API endpoint
```

## Jira Issue Hierarchy

```
Jira Epic (PROJ-100)
└── Jira Story (PROJ-103)
    ├── Jira Task (PROJ-106) - Subtask 1
    ├── Jira Task (PROJ-107) - Subtask 2
    └── Jira Task (PROJ-108) - Subtask 3
```

**Mapping to BMAD**:
- **PRD Epic** → **Jira Epic** (e.g., Epic 1 → PROJ-100)
- **PRD Story** → **Jira Story** (e.g., Story 1.1 → PROJ-103)
- **Story Task** → **Jira Task** (e.g., Task 1.1.1 → PROJ-106)

## Configuration Reference

### Core Config Jira Section

**File**: `.bmad-core/core-config.yaml`

```yaml
jira:
  enabled: auto-detect     # auto-detect from mcp.json
  projectKey: PROJ         # Your Jira project key
  baseUrl: https://your-company.atlassian.net  # Your Jira URL
  mappingFile: .bmad-core/data/jira-mapping.yaml
  issueTypes:
    epic: Epic
    story: Story
    task: Task
  defaultAssignee: null    # Optional: Jira username/email
  labels: []               # Optional: Labels for synced issues
```

**Fields**:
- `enabled`: Always use `auto-detect` (checks mcp.json)
- `projectKey`: **REQUIRED** - Your Jira project key
- `baseUrl`: **REQUIRED** - Your Jira instance URL
- `mappingFile`: Path where mapping file is stored
- `issueTypes`: Map to your Jira project's issue types
- `defaultAssignee`: Optional default assignee for created issues
- `labels`: Optional labels to add to all synced issues

## Best Practices

### 1. Sync Timing

✅ **Do**: Sync PRD after it's reviewed and approved  
✅ **Do**: Sync before starting development  
❌ **Don't**: Sync during active PRD editing

### 2. Mapping File Management

✅ **Do**: Keep mapping file with your project (or in .gitignore)  
✅ **Do**: Back up mapping file periodically  
❌ **Don't**: Manually edit mapping file (let tools manage it)

### 3. Jira as Mirror

✅ **Do**: Make changes in BMAD documents (PRD, stories)  
✅ **Do**: Re-sync after significant PRD updates  
❌ **Don't**: Edit Jira issues expecting changes to flow back to BMAD

### 4. Team Workflows

✅ **Do**: Share JIRA-SETUP.md with team members  
✅ **Do**: Use same project key across team  
✅ **Do**: Coordinate who syncs to avoid duplicates  
❌ **Don't**: Have multiple people sync same PRD simultaneously

## Troubleshooting

### Issue: "Jira MCP not found"

**Cause**: Jira MCP server not configured in `~/.cursor/mcp.json`

**Solution**: Complete setup in [JIRA-SETUP.md](JIRA-SETUP.md)

### Issue: "Project not found"

**Cause**: Wrong project key in `core-config.yaml` or no access to project

**Solution**:
1. Verify project key in Jira (check any issue key)
2. Ensure you have access to the project
3. Update `core-config.yaml` with correct key

### Issue: "Epic/Story already exists" (duplicate issues)

**Cause**: Synced multiple times without checking mapping file

**Solution**:
1. Check if `jira-mapping.yaml` exists
2. If duplicates created, manually delete in Jira
3. Use existing mapping file for subsequent syncs
4. Choose "Update existing" when prompted during sync

### Issue: Story not updating in Jira

**Cause**: Story not found in mapping file (PRD updated after sync)

**Solution**:
1. Check `jira-mapping.yaml` for story entry
2. If missing, run `*sync-to-jira` to add new stories
3. Then run `*draft` again to update story

### Issue: "Cannot create Epic issue type"

**Cause**: Your Jira project doesn't use Epic issue type

**Solution**:
1. Check Project Settings → Issue Types in Jira
2. Update `core-config.yaml` issueTypes to match your project
3. Example: Use "Feature" instead of "Epic"

## Advanced Usage

### Custom Issue Types

If your Jira project uses different issue type names:

```yaml
jira:
  issueTypes:
    epic: Feature      # Instead of Epic
    story: User Story  # Instead of Story
    task: Sub-task     # Instead of Task
```

### Multiple Projects

If working with multiple Jira projects, update `projectKey` in `core-config.yaml` per project:

```bash
# Project A
cd project-a
# Edit .bmad-core/core-config.yaml: projectKey: PROJA

# Project B  
cd ../project-b
# Edit .bmad-core/core-config.yaml: projectKey: PROJB
```

### Re-syncing After PRD Updates

If you update the PRD after initial sync:

```
@pm
*sync-to-jira

# Choose option: "Update existing issues"
# This will update Jira issues with new content from PRD
```

## FAQ

**Q: Do I need Jira to use BMAD?**  
A: No! Jira integration is completely optional. All BMAD features work without Jira.

**Q: Can changes in Jira flow back to BMAD?**  
A: No, this is one-way sync only (BMAD → Jira). BMAD documents are the source of truth.

**Q: What happens if I skip Jira sync during PRD creation?**  
A: Nothing! You can sync later anytime with `*sync-to-jira` command.

**Q: Can I use both PM and SM agents to sync?**  
A: Yes! Both agents have the `*sync-to-jira` command. Use whichever is convenient.

**Q: Will this work with Jira Server (not Cloud)?**  
A: Yes, if you use cosmix/jira-mcp which supports both Cloud and Server.

**Q: Can multiple team members sync to the same Jira project?**  
A: Yes, but coordinate to avoid duplicate issues. Share the mapping file if needed.

**Q: What if my Jira project doesn't support Epics?**  
A: Update `issueTypes` in `core-config.yaml` to use your project's issue types.
